||==================================||
   Discuz! X 轉換檔案
||==================================||

本檔案用於幫助您將原有社群產品轉換或者轉換到 Discuz! X 系列產品。
在您開始轉換之前，請務必逐條仔細閱讀以下的注意事項：

1. 轉換之前，為了確保無誤，強烈建議您備份原有資料，我們無法對轉換前沒有備份的用
   戶提供任何技術支援
2. 一般情況下，我們的轉換程式放置在發行版本的 ./utilities/convert 目錄中，此程式
   可以獨立於產品執行
3. 我們不能保障轉換或者轉換到 Discuz! X 產品當中所有的資料都是有效的，某些插件或
   者自行開發的功能資料需要您自行轉換
4. 大多數情況下，您轉換到 Discuz! X 產品以後，需要進入後台進行一系列重新設定，否
   則某些功能可能無法正常使用。
5. 除非有特殊的聲明，我們的轉換程式不會改變您原有的資料格式和設定，他會將原有資
   料重新複製一份，並做格式轉換。所以，在轉換開始之前，您應當檢查您的資料庫儲存   
   空間是否夠用。假設您原有資料是 100M，您至少需要 110M 以上的資料剩餘空間，否則
   可能無法轉換成功
6. 轉換成功後，請您刪除 convert 程式，以避免可能的安全問題

+--------------------------------------+
 轉換方法和步驟
+--------------------------------------+
1. 進入您原來的系統，關閉您的網站。
2. 進行資料備份，附件備份
3. 備份您的系統的配置檔案 例如 config.inc.php，記錄下當中的各種伺服器資料，例如
   mysql 資料庫
4. 將原有程式 全部轉移到一個備份目錄，例如 old
5. 上傳 Discuz! X 程式到您的網站目錄
6. 正確安裝 Discuz! X （如果您的 UCenter Server 是獨立安裝的，您需要提供他的存取
   路徑和創始人密碼）
7. 上傳 utilities 目錄中的 convert 資料夾到您的網站根目錄
   設定為可寫（777）
8. 瀏覽 http://您的網域名稱/convert 開始轉換/轉換
9. convert 程式當中集成了多種程式可以轉換到 Discuz! X，請您根據系統進行選擇
10. 轉換程式一般會提供一個轉換或者安裝說明，您在開始之前請仔細閱讀他
11. 按照程式提示，直至所有轉換執行完畢

+--------------------------------------+
 轉換疑難解決
+--------------------------------------+
您在轉換過程中遇到任何問題，可透過以下途徑解決

1. 到我們的討論區尋求協助和支援
   https://www.discuz.vip

2. 檢視詳細轉換指導檔案
   https://www.dismall.com

3. 商業用戶可以根據您購買的服務，使用電話、MSN、QQ、論壇等多種方式尋求技術支援


+--------------------------------------+
 常見問題和解答
+--------------------------------------+

1. 我必須要先安裝 Discuz! X 嗎？
   一般是這樣的，尤其是在進行轉換的時候，這是最關鍵的一步，否則無法轉換成功

2. 我已經安裝了 Discuz! 論壇和 UChome 兩個產品，可以轉換到 Discuz! X 產品嗎？
   可以的。Discuz! X 融合了論壇和 SNS 的功能，轉換以後，您的兩個產品將會完美整合

3. 轉換 Discuz! 和 UChome 有什麼特殊要求嗎？
   在進行轉換之前，您必須在網站上安裝 DiscuzX 程式，並保證它可以正常執行。
   如果您僅僅安裝了某一個產品，那麼轉換到 DiscuzX 是容易的，按照提示即可。
   如果您安裝了兩個或者多個產品，那麼轉換的時候要注意轉換順序，您首先應當轉換論
   壇產品到 Discuz! X，然後再執行其他轉換程式，轉換 UChome 或者 Supesite

4. 為何我轉換完畢以後，網站的用戶都沒有了
   轉換程式只有檢測到 UCenter 資料表的時候，才會轉換會員資料。如果您以前使用的是
   我們提供的 Discuz! 完整版，轉換程式可以自動檢測到 UCenter 的。但是如果您使用的
   是自己獨立安裝的 UCenter，那麼轉換程式會檢測不到。
   解決的辦法：
       請重新安裝您的 Discuz! X，安裝的時候選擇 僅安裝 Discuz! X (手動指定已經安
   裝的 UCenter Server)，然後填入您的 UCenter 的訊息，並開始安裝。安裝完畢以後重新
   開始轉換。

5. 我原來的論壇規模很大，轉換程式對資料的大小有限制嗎？是否可能確保轉換成功？
   我們的轉換程式經過多次的測試，並在 >6G 的資料環境中反覆轉換成功，因此對於原始
   資料的大小是沒有限制的，只不過隨著資料量越大，轉換時間越長。您應當保障
   資料轉換期間，伺服器一直是可用的。

6. 如果轉換中途出錯或者意外中斷，該如何處理？
   如果出現意外中斷，一般情況下您應當根據錯誤提示，排除錯誤，然後再重新開始。
   轉換程式提供了進階轉換模式，您可以選擇跳過某些轉換步驟，而不必重新來過，但您
   應當瞭解哪些步驟已經執行過，否則可能導致資料轉換不完整。
   如果您是在轉換 UChome 或者 Supesite 過程中出錯，並且此時論壇已經轉換完畢，
   那麼您可能需要重新轉換論壇，然後再執行 UChome 的轉換，否則轉換可能無法成功。
   因此，如果您需要轉換論壇 + UChome + ... 多個程式，我們建議您在每個轉換程式結束，
   對已經轉換好的資料進行備份，這樣可以在中斷的時候，恢復備份，然後繼續執行。

7. 轉換過程中需要人員干預嗎？
   除了轉換前面需要填寫一些資料和選擇轉換程式，轉換一旦開始，程式將會自動執行和
   跳轉，所以您無需進行干預。不恰當的干預反而可能在成轉換中斷或者失敗。
